# 发布检查清单

## 发布前

- [ ] `README.md` 是否清楚说明这是什么
- [ ] `INSTALL.md` 是否能指导安装
- [ ] `VERSION.md` 是否填写正确
- [ ] `CHANGELOG.md` 是否覆盖本次发布内容
- [ ] `RELEASE-NOTES.md` 是否适合作为转交说明
- [ ] `DISTRIBUTION-COPY.md` 是否可直接拿去发群 / 飞书 / 邮件
- [ ] release 目录内是否无 `.DS_Store`
- [ ] 压缩包是否重新生成

## 发布时

- [ ] 优先分享 release 压缩包
- [ ] 同时附上 `README.md` 或 `RELEASE-NOTES.md`
- [ ] 如果面对新同学，先发 `INSTALL.md`
- [ ] 如果面对负责人，优先发 `DISTRIBUTION-COPY.md` 中的短版或标准介绍

## 发布后

- [ ] 记录发给了哪些人 / 团队
- [ ] 收集反馈：最先被问到的问题是什么
- [ ] 收集反馈：最常被实际使用的是哪部分
- [ ] 收集反馈：哪些说明还不够清楚
- [ ] 为下一版 release 准备 changelog 输入
