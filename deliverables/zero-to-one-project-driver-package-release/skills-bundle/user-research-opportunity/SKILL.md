# User Research Opportunity

Use this skill to turn uncertain product direction into evidence-shaped research inputs.

## Primary Job

Produce research that changes decisions, not research that only looks complete.

## Core Rules

- Answer the most decision-relevant question first.
- Prefer realistic users, realistic tasks, and concrete context.
- Keep questions open and non-leading.
- Share preliminary findings early when they can influence direction.
- Turn findings into actionable product implications.

## Default Workflow

1. Define the decision this research should influence.
2. Identify target users and scenarios.
3. Select the lightest method that can answer the question.
4. Structure outputs into:
   - opportunity points
   - user needs
   - pain points
   - evidence or examples
   - implication for product scope
5. Update `docs/research-input.md`.

## Output Standard

Each finding should connect:

- observed issue or signal
- likely user meaning
- product implication
- open question if still uncertain

## Common Failure Modes

- Producing generic personas without evidence
- Listing methods without tying them to a decision
- Waiting too long to share preliminary findings
- Confusing broad inspiration with actionable research input
