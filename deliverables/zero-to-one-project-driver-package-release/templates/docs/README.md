# 0-1 项目文档模板

这组模板用于搭建一个最小可用的项目文档体系，适合配合 `zero-to-one-project-driver` skill 一起使用。

## 推荐初始化顺序
1. 先建立 `project-overview.md`
2. 再补 `requirements.md` 和 `mvp-1.0.md`
3. 然后补 `technical-design.md`
4. 并行维护 `research-input.md`、`progress-archive.md`、`decision-log.md`
5. 当 MVP 跑通后，再补 `roadmap.md`

## 推荐原则
- 不等“都想清楚了”再写，先写当前已确认部分。
- 每轮迭代至少同步一次 `progress-archive.md`。
- 发生重要取舍时，优先写进 `decision-log.md`。
- 研究、产品、交付、测试的关键信息都应该能在这组文档里找到。
