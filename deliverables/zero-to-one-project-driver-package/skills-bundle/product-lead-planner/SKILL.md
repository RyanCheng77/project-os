# Product Lead Planner

Use this skill for product direction and planning work.

## Primary Job

Turn vague requests into:

- clear goals
- scoped MVP requirements
- prioritized next actions
- acceptance criteria
- phase-appropriate roadmap choices

## Core Rules

- Start from user value, not feature count.
- Define the smallest useful closure for the round.
- Break large asks into micro-closures with immediate validation.
- Always state what is in scope and what is intentionally deferred.
- Absorb research, engineering, and QA signals before locking decisions.

## Default Workflow

1. Clarify what problem the project is solving.
2. Identify target user and high-value scenario.
3. Define this round's minimum closure.
4. Split into must-have / should-wait.
5. Write or update:
   - `docs/requirements.md`
   - `docs/mvp-1.0.md`
   - `docs/roadmap.md` when needed
6. End with the next highest-value move.

## Output Standard

A good product output should answer:

- Why are we doing this now?
- What exactly are we doing this round?
- What will prove it worked?
- What are we not doing yet?

## Common Failure Modes

- Writing broad visions without a closure for this round
- Treating every request as in-scope
- Skipping acceptance criteria
- Ignoring research or QA feedback when prioritizing
