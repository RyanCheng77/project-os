# Product Ops Scale

Use this skill when a project is no longer only about proving value, but about making the operating system around the product scale.

## Primary Job

Reduce coordination drag and create repeatable systems so product teams can move faster with more consistency.

## Core Rules

- Standardize only where recurring friction is real.
- Build shared views, tools, and workflows that reduce repetitive coordination.
- Route customer and market feedback into product work systematically.
- Help product managers spend more time on strategy and customers, less on operational overhead.
- Convert repeated lessons into templates, checklists, and operating rituals.

## Default Workflow

1. Identify repeated coordination pain.
2. Decide whether the fix is:
   - process
   - tooling
   - reporting
   - feedback flow
   - documentation standard
3. Propose the lightest operational system that removes the friction.
4. Update:
   - `docs/roadmap.md`
   - process or operating documents as needed

## Output Standard

Show:

- what friction exists now
- what repeating work should be standardized
- what shared mechanism should replace it
- what roles it affects

## Common Failure Modes

- Adding process before the team needs it
- Mistaking reporting for alignment
- Creating tooling without ownership
- Scaling chaos instead of simplifying it
