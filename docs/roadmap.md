# 路线图

## 阶段 1：结构收口

目标：

- 明确主维护源
- 补齐仓库入口
- 补齐项目文档
- 说明 deliverables 之间关系

当前状态：

- 进行中

## 阶段 2：独立工具包索引化

目标：

- 为每个 deliverable 建立统一入口
- 明确适用场景、输入、输出、依赖和维护状态
- 让外部读者无需读完整仓库也能选工具

预计动作：

- 新增 `deliverables/README.md`
- 为每个工具包加状态标签，例如：`draft`、`stable`、`enterprise-derived`

## 阶段 3：发布流程标准化

目标：

- 降低 release 目录与源码目录手工同步成本
- 形成可重复发布流程

预计动作：

- 补 release 生成脚本
- 补校验清单自动化
- 统一版本号与 changelog 更新方式

## 阶段 4：业务副本治理

目标：

- 控制业务项目中的复制件漂移
- 明确何时复制、何时回流

预计动作：

- 给业务副本增加来源说明
- 设计最小同步规范

## 阶段 5：公开化准备

目标：

- 让工具包具备更强的可阅读性和可分享性
- 为后续对内大范围分发或对外整理打基础

预计动作：

- 增加示例接入案例
- 增加常见问题和最佳实践
- 增加首屏导览和上手路径
