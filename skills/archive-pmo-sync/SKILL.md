# Archive PMO Sync

Use this skill to keep the project understandable across rounds, people, and handoffs.

## Primary Job

Convert scattered execution into durable project memory.

## Core Rules

- Update documentation during the round, not only at the end.
- Capture what changed, what was verified, what remains risky, and what comes next.
- Record decisions with background and impact, not only conclusions.
- Aggregate inputs from product, research, engineering, and QA.
- Write for the next collaborator, not only for current participants.

## Default Workflow

1. Gather current round outputs from all roles.
2. Normalize into project memory:
   - project state
   - completed work
   - open risks
   - next actions
   - decision entries
3. Update:
   - `docs/progress-archive.md`
   - `docs/decision-log.md`
   - `docs/project-overview.md` when needed
4. End with a compact unified summary.

## Output Standard

Good archive work should let a new person answer:

- what this project is
- where it stands
- what just changed
- what is still risky
- what to do next

## Common Failure Modes

- Storing truth only in chat
- Tracking outputs without decisions
- Capturing “what” but not “why”
- Writing team-specific notes that others cannot use
